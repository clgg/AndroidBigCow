# 项目实战场景题

## Q: 如果 App 首页冷启动很慢，你会如何排查和优化？

标签：场景题 / 高频 / 性能

复习状态：未掌握

考察点：

- 指标定义。
- Trace 分析。
- 初始化治理。

答案要点：

- 先定义指标：冷启动、首帧、可交互、首页数据可见时间。
- 用 Perfetto、启动日志、方法耗时和线上监控定位耗时段。
- 检查 Application、ContentProvider、首屏布局、图片、主线程 IO、同步网络、类加载。
- 优化应分层推进：延迟非必要初始化、异步化并限流、首屏轻量化、基线配置文件和缓存预热。

深挖追问：

- 异步初始化如何保证依赖顺序？
- 启动优化如何评估收益？
- 首帧快但首页数据慢怎么处理？

常见误区：

- 只用 System.currentTimeMillis 打日志，不看系统 trace。

## Q: 大型列表滑动卡顿，你会如何定位？

标签：场景题 / 高频 / 性能

复习状态：未掌握

考察点：

- RecyclerView。
- 图片加载。
- 主线程耗时。

答案要点：

- 先复现并用 Perfetto、FrameMetrics 或自研卡顿监控定位掉帧区间。
- 检查 onCreateViewHolder、onBindViewHolder、DiffUtil、嵌套滑动、布局层级和图片加载。
- 图片要做尺寸裁剪、请求取消、占位、缓存和生命周期绑定。
- 对复杂 item 可做局部刷新、预加载、异步 Diff、共享 RecycledViewPool。

深挖追问：

- payload 如何避免全量绑定？
- 列表中视频自动播放如何治理性能？
- 如何监控线上列表卡顿？

常见误区：

- 只减少 item 数量，不分析真正耗时。

## Q: 设计一个图片加载库，需要考虑哪些模块？

标签：场景题 / 架构 / 图像

复习状态：未掌握

考察点：

- 请求生命周期。
- 缓存。
- 解码。

答案要点：

- 核心模块包括请求构建、生命周期绑定、内存缓存、磁盘缓存、网络加载、解码变换、占位和错误处理。
- 列表场景要处理 View 复用、请求取消、目标尺寸、优先级和并发控制。
- 缓存 key 要考虑 URL、尺寸、变换参数和格式。
- 大图、GIF、WebP、硬件位图和低内存策略都要按场景处理。

深挖追问：

- 内存缓存和磁盘缓存如何协作？
- 如何避免图片错位？
- 如何支持圆角、模糊、水印等变换？

常见误区：

- 只实现网络下载和 ImageView 设置，不处理生命周期和缓存。

## Q: 登录态失效后，多接口同时返回 401，如何设计 token 刷新？

标签：场景题 / 网络 / 高频

复习状态：未掌握

考察点：

- 并发控制。
- 请求重放。
- 错误边界。

答案要点：

- 多请求同时 401 时应保证只有一个刷新任务，其余请求等待结果或快速失败。
- 刷新成功后可重放原请求，刷新失败则清理登录态并通知 UI。
- 要避免在 OkHttp 拦截器中递归调用同一个 client 导致死锁。
- 需要处理刷新接口也失败、用户主动退出、token 被覆盖、多进程等边界。

深挖追问：

- 请求等待刷新时如何避免阻塞过久？
- 刷新 token 存储如何保证一致性？
- WebSocket 或长连接如何同步登录态？

常见误区：

- 每个 401 都刷新一次 token。
- 刷新失败后仍然重试导致请求风暴。

## Q: 线上出现偶发 ANR，你会如何推进排查？

标签：场景题 / ANR / 高频

复习状态：未掌握

考察点：

- 现场保留。
- 堆栈分析。
- 系统化定位。

答案要点：

- 先收集 ANR 类型、机型系统、前后台状态、主线程堆栈、CPU/IO/内存状态和业务日志。
- 分析主线程是否在 IO、锁等待、Binder 调用、布局绘制、死循环或等待异步结果。
- 如果堆栈不稳定，要增加关键链路耗时埋点和卡顿监控。
- 修复后通过压测、弱网、低端机和回归监控确认效果。

深挖追问：

- 主线程堆栈显示 Binder transact 阻塞怎么办？
- 线上没有 trace 文件时如何补充监控？
- ANR 和卡顿监控如何关联？

常见误区：

- 看不到本地复现就无法推进。
- 不区分输入、广播、服务、ContentProvider 不同 ANR 场景。
