## Q: HTTPS 为什么安全？TLS 握手大致做了什么？

标签：高频 / 网络 / 安全

复习状态：未掌握

考察点：

- 加密。
- 证书。
- 身份认证。

答案要点：

- HTTPS 在 HTTP 之下加入 TLS，提供加密传输、完整性校验和身份认证。
- 握手阶段会协商协议版本、加密套件，校验证书，并生成会话密钥。
- 非对称加密主要用于身份认证和密钥协商，对称加密用于后续数据传输。
- 客户端必须正确校验证书链、域名和有效期。

深挖追问：

- 证书固定是什么？
- 中间人攻击如何发生？
- HTTP/2 和 HTTPS 是什么关系？

常见误区：

- 认为 HTTPS 只是简单加密。
- 自定义 TrustManager 时跳过所有证书校验。

## Q: OkHttp 拦截器有哪些类型？适合做什么？

标签：高频 / 网络 / Android

复习状态：未掌握

考察点：

- Application Interceptor。
- Network Interceptor。
- 请求链路。

答案要点：

- Application Interceptor 位于应用层，可处理重试、统一 Header、日志、业务错误。
- Network Interceptor 更接近网络层，可观察重定向、缓存和实际网络请求。
- 拦截器应避免做长时间阻塞和复杂业务耦合。
- 统一鉴权刷新 token 时要注意并发请求和死锁风险。

深挖追问：

- token 过期多请求并发刷新如何处理？
- OkHttp 连接池如何复用连接？
- 缓存拦截器如何工作？

常见误区：

- 在拦截器里随意发同步请求导致阻塞或递归。

## Q: SQLite 索引为什么能提升查询？什么时候会失效？

标签：存储 / 高频 / 性能

复习状态：未掌握

考察点：

- B-Tree。
- 查询条件。
- 执行计划。

答案要点：

- 索引通过额外数据结构减少扫描范围，常见实现基于 B-Tree。
- 对高选择性字段、排序字段、关联字段建立合适索引可以提升查询。
- 函数包裹列、类型不匹配、前导模糊匹配、组合索引不满足最左前缀等可能导致索引失效。
- 索引会增加写入和存储成本，不是越多越好。

深挖追问：

- 如何查看 SQLite 查询计划？
- 组合索引顺序怎么设计？
- Room 中如何声明索引？

常见误区：

- 所有字段都建索引。
- 只看单次查询，不考虑写入成本。

## Q: SharedPreferences 有哪些问题？DataStore 解决了什么？

标签：存储 / Jetpack / 易混

复习状态：未掌握

考察点：

- 同步 IO。
- 数据一致性。
- 协程。

答案要点：

- SharedPreferences 使用简单，但可能涉及主线程 IO、apply 异步落盘时机、跨进程不可靠等问题。
- DataStore 基于协程和 Flow，提供异步、事务式更新和更好的错误处理。
- Preferences DataStore 适合简单键值，Proto DataStore 适合强类型结构化数据。
- DataStore 不适合大量关系型数据，这类场景仍应使用数据库。

深挖追问：

- apply 和 commit 有什么区别？
- DataStore 如何处理并发写？
- Proto DataStore schema 如何演进？

常见误区：

- 把 DataStore 当数据库用。

## Q: Android 本地敏感数据如何保护？

标签：安全 / 场景题 / 高频

复习状态：未掌握

考察点：

- 加密存储。
- KeyStore。
- 威胁模型。

答案要点：

- 先明确威胁模型：防普通文件泄漏、Root 环境、调试抓取、备份泄漏的策略不同。
- 敏感数据应尽量减少本地存储，必要时使用 Android Keystore 管理密钥。
- 可结合 EncryptedSharedPreferences、加密数据库或业务加密方案。
- 日志、剪贴板、备份、截图和 WebView JS Bridge 也可能泄露敏感信息。

深挖追问：

- Keystore 里的密钥能否导出？
- token 应该存哪里？
- Root 设备上能保证绝对安全吗？

常见误区：

- 只对字符串做 Base64 就认为安全。
- 忽略日志和备份通道。
